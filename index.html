Frequently Asked Questions

1. What is the difference between var, let, and const?
var: Function-scoped, can be redeclared/updated.
let: Block-scoped, can be updated but not redeclared.
const: Block-scoped, cannot be updated or redeclared.
2. What is the difference between map(), forEach(), and filter()?
map(): creates a new array by transforming each element.
forEach(): executes a function for each item without returning anything.
filter(): returns a new array with only elements that pass a condition.
3. Explain arrow functions and how they are different from regular functions?
Arrow functions (=>) are shorter and inherit this from their surroundings, while regular functions (function) have their own this and support constructors/arguments. Use arrows for simplicity, regular functions when you need dynamic this.
4. How JavaScript Promises work?
JavaScript Promises handle async tasks by tracking success (.then()) or failure (.catch()). They start as pending, then resolve to fulfilled (success) or rejected (error)—cleaning up callback chaos.
5. How closures work in JavaScript?
Closures in JavaScript allow a function to remember and access its outer scope’s variables even after that outer function has finished executing. They "close over" (capture) the environment where they were created.
